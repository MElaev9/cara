# Бот «Караван» — Telegram-бот для расчёта банкетных закупок

## Описание

Telegram-бот для автоматического расчёта количества продуктов для банкетов, свадеб, корпоративов и других мероприятий.

### Возможности (MVP)

- Создание мероприятия (название + количество гостей)
- Выбор блюд из каталога (22 блюда, 5 категорий)
- Автоматический расчёт ингредиентов с учётом числа гостей
- Объединение одинаковых ингредиентов из разных блюд
- Автоматическое добавление запаса **+7%**
- Архив всех мероприятий с карточкой закупки

## Запуск

### Локально (без Docker)

```bash
# 1. Установить зависимости
pip install -r requirements.txt

# 2. Задать токен бота
export BOT_TOKEN=ваш_токен_от_BotFather

# 3. Запустить
python bot.py
```

### Через Docker Compose

```bash
# Создать файл .env
echo "BOT_TOKEN=ваш_токен_от_BotFather" > .env

# Запустить
docker-compose up -d
```

### Получить токен бота

1. Открыть [@BotFather](https://t.me/BotFather) в Telegram
2. Написать `/newbot`
3. Следовать инструкциям
4. Скопировать полученный токен

## База данных

SQLite-файл `karavan.db` (или путь из `DB_PATH`).

### Таблицы

| Таблица | Описание |
|---|---|
| `dishes` | Блюда (id, name, category, serves) |
| `ingredients` | Ингредиенты блюд (dish_id, name, amount, unit) |
| `events` | Мероприятия (id, name, guests, dish_ids, created_at) |

---

## Алгоритм расчёта

1. Для каждого выбранного блюда: `multiplier = guests / serves`
2. Каждый ингредиент: `amount = amount_per_recipe × multiplier`
3. Одинаковые ингредиенты суммируются
4. К итогу добавляется **7% запас**: `amount × 1.07`
5. Граммы ≥1000 → переводятся в кг; миллилитры ≥1000 → в литры

### Коэффициенты подачи (поле `serves` в рецепте)

| Категория | Порций на N гостей |
|---|---|
| Салаты | 3 |
| Закуски | 4 |
| Горячее | 1 |
| Гарниры | 1 |
| Десерты | 1 |
